# Universal Clean UI Skill

## Purpose

Generate interfaces that prioritize clarity, usability, visual hierarchy, consistency, and simplicity.

The goal is not to create visually impressive designs, but interfaces that feel intuitive, organized, modern, and effortless to use.

---

# Core Philosophy

A clean interface removes friction.

Every element must have a purpose.

Before adding any component, ask:

> Does this help the user complete a task or understand information?

If not, remove it.

---

# Design Principles

## Clarity Over Decoration

Prioritize understanding over aesthetics.

Avoid:

- Decorative elements without purpose
- Excessive illustrations
- Unnecessary icons
- Visual noise
- Complex patterns

Every visual decision must improve comprehension.

---

## Simplicity Over Density

Reduce the number of visible elements whenever possible.

Prefer:

- Fewer actions
- Fewer colors
- Fewer components
- Fewer visual styles

A user should understand a screen within seconds.

---

## Visual Hierarchy

Information must communicate importance through:

- Size
- Weight
- Position
- Spacing
- Contrast

The most important content should attract attention first.

The least important content should fade into the background.

---

## Consistency

Identical elements must behave and appear identically.

Maintain consistency in:

- Spacing
- Typography
- Colors
- Component structure
- Interaction patterns

Users should never have to relearn the interface.

---

## Whitespace Is Functional

Empty space is not wasted space.

Whitespace should:

- Separate groups
- Improve readability
- Reduce cognitive load
- Create visual balance

Avoid crowded layouts.

---

## Progressive Disclosure

Show only what the user needs right now.

Hide advanced options until necessary.

Avoid presenting all information simultaneously.

---

## Content First

Design should support content.

Content should never adapt to decorative design choices.

The interface exists to communicate information and enable actions.

---

# Layout Rules

## Clear Structure

Interfaces should be organized into logical sections.

Each section should represent:

- A goal
- A task
- A category of information

Users should instantly understand where they are.

---

## Predictable Navigation

Navigation should be obvious.

Users should always know:

- Where they are
- Where they can go
- How to return

Avoid surprising interactions.

---

## Logical Grouping

Related elements should appear together.

Unrelated elements should be visually separated.

Use spacing before adding borders.

---

# Typography Rules

## Typography Creates Hierarchy

Use typography to establish importance.

Hierarchy should be visible without reading.

Typical structure:

- Primary heading
- Secondary heading
- Supporting content
- Metadata

Avoid excessive font variations.

---

## Readability First

Text should be easy to scan.

Avoid:

- Long dense paragraphs
- Extremely small text
- Excessive font weights
- Decorative typography

---

# Color Rules

## Use Color With Intention

Color should communicate:

- Importance
- Status
- Feedback
- Actions

Not decoration.

---

## Limit Visual Competition

Most of the interface should remain visually neutral.

Accent colors should draw attention only to important actions or information.

Avoid multiple competing focal points.

---

# Component Rules

## Components Must Have Clear Purpose

Every component should answer:

- What information does it provide?
- What action does it enable?

If neither answer exists, reconsider its existence.

---

## Prioritize Recognition

Users should recognize components immediately.

Avoid reinventing common patterns.

Use familiar interaction models whenever possible.

---

# Interaction Rules

## Reduce Cognitive Load

Users should think about their task.

Not about the interface.

Reduce:

- Decision fatigue
- Visual complexity
- Unnecessary interactions

---

## Minimize Steps

Allow users to achieve goals with the fewest reasonable interactions.

Avoid unnecessary confirmations and intermediate screens.

---

## Feedback Is Mandatory

Every user action should provide feedback.

Users should always know:

- What happened
- What is happening
- What will happen next

---

# Accessibility Principles

Interfaces should be usable by as many people as possible.

Ensure:

- Sufficient contrast
- Clear labels
- Predictable behavior
- Readable typography
- Accessible touch targets

Accessibility is part of good design, not an optional enhancement.

---

# Universal Evaluation Checklist

Before considering a UI complete, verify:

- Is every element necessary?
- Is the hierarchy immediately obvious?
- Can users understand the screen within seconds?
- Are related items grouped together?
- Is spacing creating clarity?
- Is navigation predictable?
- Is content prioritized correctly?
- Is visual noise minimized?
- Are interactions obvious?
- Is the interface consistent?

If any answer is no, simplify further.

---

# Golden Rule

A clean UI is not defined by minimalism.

A clean UI is defined by the absence of unnecessary complexity.
